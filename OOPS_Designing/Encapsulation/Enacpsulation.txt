1. Definition : Binding / Wrapping data members and members functions together into a single unit is called Encapsulation.

2. Functions of Encapsulation : 
    (a): It provides a secure layer
    (b): Hides internal implementation of code and data in a class
    (c): Exposes only necessary information to the external world

3. As we are hiding data from outer classes or the world, Encapsulation is also known as Data hiding.

4. The goal is to implement classes in a way that prevents unauthorize access to or modification of the original contents of a class
   by its instances (or objects).

5. Perfect Encapsulation : If all data members / variables are private. It can be access by the help of Getters and Setters.

Access Modifiers: It Define the Scope of class / variable or methods

(a) : Public Access Modifiers: Members Accessible from any part of the program.
                               They can be accessed by objects of the class and external code.

(b) : Private Access Modifiers: Members Accessible only within the same class.
                                They are not accessible from outside the class, including derived class.

(c) : Protected Access Modifiers: Members Accessible within the same class and by derived classes.
                               1  They are not dirctly accessible from external code.
